# UI Design System

Apply these UI design standards strictly to create a professional SaaS/company website look and feel.

## Goal

Make the UI look like a professional SaaS/company website.

## Layout Standards

- Use mobile-first responsive design.
- Use Flexbox and Grid appropriately based on layout needs.
- Keep layouts clean and avoid visual clutter.
- Prioritize spacing, hierarchy, and alignment for readability.

## Interaction Standards

- Add hover effects for interactive elements, especially buttons and cards.
- Use smooth transitions with Tailwind utilities such as:
  - `transition-all`
  - `duration-200`
- Keep interaction feedback subtle, consistent, and fast.

## Color Standards

- Use a consistent color palette across pages and components.
- Avoid using too many different colors.
- Keep contrast and legibility strong for text and controls.

## Design Principles

- Prefer consistency over novelty.
- Reuse UI patterns for repeated sections and components.
- Keep visual weight balanced: clear sections, predictable spacing, and simple composition.

## Implementation Checklist

Use this checklist for UI tasks:

```markdown
UI Design Checklist
- [ ] Mobile-first responsive layout is applied
- [ ] Flex/Grid usage matches the layout purpose
- [ ] UI is clean and not cluttered
- [ ] Buttons and cards include hover effects
- [ ] Transitions use `transition-all` and `duration-200` (or equivalent smooth settings)
- [ ] Color palette is consistent across the UI
- [ ] Color usage is restrained and professional
- [ ] Final UI reflects a SaaS/company-grade visual quality
```
